# uvlab

[English](./README.md) | 日本語

`uv` のパワーを最大限に活用した、データサイエンティストのためのポータブルな JupyterLab テンプレート。
環境構築に時間を溶かすのをやめ、解析に集中するためのツールです。

## 特徴

* **1ファイル、1環境**: 各 `lab/*.py` が「依存ライブラリ」と「Jupyter設定」を両方保持。ファイル1つで環境が完結します。
* **Many-to-Many の柔軟性**: 1つの環境（lab）で複数のコンペ（notebooks）を切り替えたり、1つのコンペに対して異なる環境を使い分けることが可能です。
* **設定不要のポータビリティ**: `pathlib` による動的パス解決により、リポジトリをどこにクローンしても即座に動作します。
* **クリーンな運用**: `uv run` による自動環境構築と、`Ctrl+C` による確実なプロセス終了。

## 使い方

### 1. 準備

[uv](https://github.com/astral-sh/uv) がインストールされていることを確認してください。

### 2. 起動

`lab/` ディレクトリ内のスクリプトを直接実行します。

```bash
# デフォルト（notebooks/ 直下）で起動
uv run lab/standard.py

# 特定のコンペ/プロジェクト用フォルダを指定して起動
uv run lab/standard.py titanic

```

### 3. オプション

コマンドライン引数で動的に設定を変更できます。

```bash
# ポート番号を変更して起動
uv run lab/lab_standard.py --port 9000

# ブラウザを自動で開かずに起動
uv run lab/lab_standard.py --no-browser

```

## 新しい解析環境の追加

1. `lab/sample.py` をコピーして新しいファイル（例: `lab/pytorch.py`）を作成します。
2. ファイル冒頭の `dependencies` セクションに必要なライブラリを記述します。
3. `if __name__ == "__main__":` ブロックの `launch(default_competition="...")` を必要に応じて書き換えます。

## ディレクトリ構成

* `lab/`: 環境定義スクリプト群（各コンペ/講座のエントリーポイント）
* `notebooks/`: すべての `.ipynb` ファイルが保存される作業スペース。コンペごとにサブフォルダが自動生成されます。

## 注意事項

* **ブラウザ利用の推奨**: VSCode の Jupyter 拡張機能ではなく、ブラウザ（`http://localhost:8888/lab`）での利用を推奨します。これにより、Jupyter の全機能と安定したプロセス管理が保証されます。
